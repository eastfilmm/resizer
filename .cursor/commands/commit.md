# 커밋 자동화 명령어

코드 변경 작업 완료 후 자동으로 Git 커밋을 생성합니다.

## 작업 순서

1. 변경된 파일 확인 (`git status`)
2. 변경 사항 분석 및 주요 변경 내용 파악
3. 변경된 모든 파일 스테이징 (`git add`)
4. 의미 있는 커밋 메시지로 커밋 생성 (`git commit`)

## 커밋 메시지 규칙

- 작업 내용을 명확하게 요약
- 한국어로 작성 (프로젝트 컨벤션)
- 변경된 주요 파일/기능 언급
- 예시:
  - `feat: ImageCanvas에 Safari 최적화 적용`
  - `fix: 사이트맵 URL 오류 수정`
  - `refactor: 환경 변수 기반 base URL 유틸리티 추가`
  - `docs: AGENTS.md에 커밋 자동화 섹션 추가`

## 주의사항

- 커밋 전에 변경사항을 확인 (`git status`, `git diff`)
- 민감한 정보가 포함된 파일은 커밋하지 않음 (`.env`, 민감한 키 등)
- 린트 에러가 있는 경우 커밋 전에 수정 권장
- `.gitignore`에 포함된 파일은 자동으로 제외됨

## 사용 예시

```
변경사항 커밋해줘
```

또는

```
커밋 자동화로 커밋 생성해줘
```
