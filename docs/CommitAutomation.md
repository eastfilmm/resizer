**역할**: 코드 변경 작업 완료 후 자동으로 Git 커밋 생성

**작업 내용**:
1. 변경된 파일 확인 (`git status`)
2. 변경 사항 분석 및 AGENTS.md 최신화
3. 분석 내용 기반 커밋 메시지 자동 생성
3. `git add` 실행 (변경된 모든 파일)
4. 의미 있는 커밋 메시지로 `git commit` 실행

**사용 방법**:
```
커밋 자동화 서브 에이전트로 커밋 생성해줘
```

또는

```
커밋 자동화: 변경사항 커밋해줘
```

**커밋 메시지 규칙**:
- 작업 내용을 명확하게 요약
- 한국어 또는 영어로 작성 (프로젝트 컨벤션에 따라)
- 변경된 주요 파일/기능 언급
- 예: "feat: ImageCanvas에 Safari 최적화 적용", "docs: AGENTS.md에 커밋 자동화 섹션 추가"

**주의사항**:
- 커밋 전에 변경사항을 확인 (`git status`, `git diff`)
- 민감한 정보가 포함된 파일은 커밋하지 않음 (`.env`, 민감한 키 등)
- 린트 에러가 있는 경우 커밋 전에 수정 권장
